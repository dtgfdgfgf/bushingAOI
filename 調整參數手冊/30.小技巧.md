## 快速設定技巧

### 💡 技巧 1：快速複製相似料號

如果您有多個非常相似的料號要設定：

```
1. 設定第一個料號（完整流程）
2. 匯出工作階段：
   檔名：param_session_料號A.json

3. 複製該檔案並重新命名：
   param_session_料號B.json

4. 用記事本編輯 料號B.json：
   - 將 "targetType" 改為 料號B
   - 微調不同的參數值

5. 開啟料號B的參數設定視窗
6. 匯入 料號B.json
7. 只調整差異的參數
8. 儲存
```

**注意**：這種方法只適用於極度相似的料號！

---

### 💡 技巧 2：建立參數範本

為不同產品類型建立參數範本：

```
產品類型A（小型精密件）：
- exposure = 3000
- gain = 8
- threshold範圍：0.3-0.5
- objBias = 10

產品類型B（大型鑄件）：
- exposure = 6000
- gain = 12
- threshold範圍：0.5-0.7
- objBias = 30

產品類型C（透明件）：
- exposure = 2000
- gain = 6
- 需要白平衡校準
- threshold範圍：0.2-0.4
```

將這些範本記錄在筆記本中，設定或調整料號參數時可快速參考。

---

### 💡 技巧 3：善用「部分自動導入」

**最佳使用時機**：
```
✅ 料號與參考料號非常相似
✅ 只有尺寸或PTFE顏色微調
✅ 想快速設定條件式參數（如油溝相關）
```

**最佳實踐流程**：
```
1. 手動設定相機參數
2. 完成位置參數校準
3. 點擊「部分自動導入」快速設定條件式參數
4. 手動微調其他檢測參數
5. 設定時間參數
6. 完整測試驗證
```

## 參數調整技巧

### 💡 技巧 4：批次調整閾值

如果您發現所有檢測區域的閾值都需要調整：

```
1. 記錄調整幅度（例如：全部減少 0.1）
2. 在「檢測參數」的「已新增已修改區」
3. 找到所有閾值參數：
   threshold1 = 0.5 → 0.4
   threshold2 = 0.6 → 0.5
   threshold3 = 0.55 → 0.45
4. 逐一修改
5. 標記為已修改並儲存
6. 重新測試驗證
```

**小技巧**：可以用 Excel 輔助計算

```
1. 匯出工作階段為 JSON
2. 複製參數到 Excel
3. 使用公式批次計算：=A2-0.1
4. 複製結果回參數設定視窗
```

---

### 💡 技巧 5：利用對照表快速排錯

建立常見問題對照表：

| 症狀 | 可能原因 | 調整參數 | 調整方向 |
|------|---------|---------|---------|
| 影像太暗 | 曝光不足 | exposure | +500~1000 |
| 影像太亮 | 曝光過度 | exposure | -500~1000 |
| 影像模糊 | 曝光時間太長 | exposure | -200~500 |
| 雜訊太多 | 增益太高 | gain | -2~5 |
| 位置偏移 | objBias太小 | objBias | +5~10 |
| 大量誤判 | 閾值太嚴 | threshold | +0.05~0.1 |
| 大量漏檢 | 閾值太寬 | threshold | -0.05~0.1 |

### 💡 技巧 6：參數版本管理

在調整參數時，建立版本記錄：

```
料號：10215112360T

版本 v1.0（2024-01-15）
- exposure = 5000
- threshold1 = 0.5
- 準確率：92%
- 問題：誤判較多

版本 v1.1（2024-01-16）
- exposure = 5500（提高亮度）
- threshold1 = 0.55（放寬閾值）
- 準確率：96%
- 問題：已解決

版本 v1.2（2024-01-20）
- threshold1 = 0.52（微調）
- 準確率：98%
- ✅ 正式上線版本
```

**工具**：使用 Git 或簡單的文字檔記錄

## 效率提升技巧

### 💡 技巧 7：建立測試樣品庫

為每個料號準備專用的測試樣品：

```

├─ 良品/
│  ├─ good_001.jpg
│  ├─ good_002.jpg
│  └─ ... (共 10 張)
├─ 不良品/
│  ├─ defect_scratch_001.jpg（刮痕）
│  ├─ defect_hole_001.jpg（孔洞）
│  ├─ defect_crack_001.jpg（裂痕）
│  └─ ... (每種缺陷 2-3 張)
└─ 邊界案例/
   ├─ borderline_001.jpg（接近不良但可接受）
   └─ borderline_002.jpg
```

**好處**：
- 快速驗證參數調整效果
- 標準化測試流程
- 新人訓練使用

## 故障排除技巧

### 💡 技巧 10：建立故障排除檢查清單

當遇到問題時，系統化檢查：

#### **影像問題檢查清單**
```
□ 相機連線正常？
□ 照明設備開啟？
□ 照明位置正確？
□ 相機參數（exposure, gain）合理？
□ delay 時間適當？
□ 產品放置正確？
```

#### **檢測問題檢查清單**
```
□ 位置參數已校準？
□ objBias 設定適當？
□ 閾值範圍合理？
□ AI 模型檔案存在？
□ 測試樣品是標準樣品？
□ 照明穩定？
```

#### **系統問題檢查清單**
```
□ PLC 連線正常？
□ 資料庫未被鎖定？
□ 系統版本正確？
□ 權限設定正常？
□ 硬碟空間足夠？
```

---

### 💡 技巧 11：使用系統日誌排查問題

當遇到錯誤時：

```
1. 檢查主程式日誌：
   .\logs\peilin_log-{日期}.txt

2. 檢查相機日誌：
   bin\x64\Release\CameraLog_{日期}.txt
   bin\x64\Release\CameraWarning_{日期}.txt

3. 搜尋關鍵字：
   - "錯誤"
   - "失敗"
   - "Exception"
   - 料號編號

4. 記錄錯誤訊息
5. 聯絡技術支援時提供日誌
```

---

### 💡 技巧 12：備份與復原策略

**定期備份**：
```
每週備份：
1. 匯出所有料號的工作階段
2. 備份資料庫檔案（mydb.sqlite）
3. 備份 AI 模型檔案

儲存位置：
- 本機：D:\Backup\{日期}\
- 網路：\\ServerName\Backup\AOI\{日期}\
```

**快速復原**：
```
如果參數設定錯誤：
1. 找到最近的備份
2. 匯入工作階段 JSON 檔案
3. 或直接替換資料庫檔案（需停止主程式）
```

## 實戰案例

### 案例 1：快速設定同系列產品

**情境**：有 5 個尺寸略有不同的同系列產品要設定

**策略**：
```
1. 詳細設定第一個產品（花費 2 小時）
2. 匯出工作階段
3. 複製 JSON 檔案 4 次
4. 用文字編輯器批次修改：
   - 料號編號
   - OD 相關參數
   - delay 時間（如果速度不同）
5. 逐一匯入並微調（每個 15 分鐘）

總時間：2 小時 + 4 × 15 分鐘 = 3 小時
（比逐一設定節省 5 小時）
```

---

### 案例 2：排查間歇性誤判問題

**情境**：良品偶爾被誤判為不良品

**排查步驟**：
```
1. 記錄誤判發生的時間和頻率
2. 檢查日誌找出規律：
   - 特定時間段？（照明變化）
   - 特定樣品？（產品差異）
   - 隨機發生？（系統不穩定）

3. 根據規律調整：
   情況A：特定時間段
   → 檢查照明穩定性
   → 重新白平衡校正

   情況B：特定樣品
   → 檢查 objBias 是否太小
   → 調整閾值容許度

   情況C：隨機發生
   → 檢查系統硬體
   → 降低閾值敏感度
```
