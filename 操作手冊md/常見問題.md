## 介面操作問題

### ❓ Q1：參考區的參數消失了怎麼辦？

**A：參考區的參數永遠不會消失**

如果您看不到參考區的參數，可能是：

**原因1：沒有選擇來源料號**
```
解決方法：
1. 檢查視窗上方「來源料號」下拉選單
2. 確認有選擇一個料號
3. 如果是空白，重新選擇來源料號
```

**原因2：來源料號沒有參數**
```
解決方法：
請聯絡工程師
```

**原因3：視窗沒有正確載入**
```
解決方法：
1. 關閉參數設定視窗
2. 重新開啟
```

---

### ❓ Q2：為什麼我編輯了參數，但儲存後沒有變化？

**A：可能是參數還在「已新增未修改區」**

**問題原因**
```
只有「已新增已修改區」的參數會被儲存到資料庫！
```

**解決方法**
```
1. 確認參數在「已新增已修改區」
2. 如果參數在「已新增未修改區」：
   - 勾選該參數
   - 點擊「標記為已修改 →」
   - 再點擊「儲存此分類」
```

---

### ❓ Q3：位置校準後，參考區的位置參數變成空白了？

**A：這是舊版本的 bug，已在新版本修正**

**如果您使用的是新版本**
```
→ 參考區的參數不會消失
→ 校準結果會直接新增到「已新增已修改區」
```

**如果您使用的是舊版本**
```
解決方法：
1. 校準前先複製位置參數到「已新增未修改區」
2. 然後再開啟校準工具
3. 校準完成後，參考區的參數會保留
```

---

### ❓ Q4：objBias 參數找不到在哪裡？

**A：objBias 在「檢測參數」分類中**

```
正確位置：
[檢測參數] 標籤 → 參考區 → 找到 objBias1, objBias2 等參數

常見誤解：
❌ objBias 不在「位置參數」分類中
   （雖然它和位置有關，但它是用來調整檢測容許度的）
```

---

### ❓ Q5：進度條顯示 100% 但仍有分類未完成？

**A：可能是進度計算的視覺誤差**

**檢查方法**
```
1. 查看狀態列的分類圖示：
   ✅ 相機(3/3)   ← 已完成
   ⚠️ 位置(5/8)   ← 進行中
   🔧 檢測(需校準) ← 需要外部工具
   ❌ 時間(0/4)   ← 未開始

2. 逐一檢查每個分類：
   - 點擊分類標籤
   - 查看「已新增已修改區」是否有參數
   - 確認已點擊「儲存此分類」

3. 如果某個分類確實完成但圖示錯誤：
   - 關閉視窗重新開啟
   - 或忽略圖示，直接測試驗證
```

## 參數設定問題

### ❓ Q6：調整了 exposure 但影像還是太暗怎麼辦？

**可能原因與解決方法**：

**原因1：照明不足**
```
解決方法：
- 檢查照明設備是否正常
❌ 請勿改動電源電壓 
```

**原因2：產品本身顏色很深**
```
解決方法：
- exposure 可以調更高
- 搭配 gain 微調
```

**原因3：相機設定問題**
```
解決方法：
- 檢查相機硬體連線
- 聯絡工程師
```

---

### ❓ Q7：影像清晰但總是偏左或偏右？

**可能原因**：這是 delay 的問題，不是 exposure/gain

```
解決方法：
1. 產品偏左（還沒到位） → delay 增加
2. 產品偏右（已經過頭） → delay 減少
3. 每次調整 10-20 ms
4. 重複測試直到居中
```

---

### ❓ Q8：影像一下亮一下暗，不穩定？

**可能原因**：

**原因1：照明不穩定**
```
解決方法：
- 檢查照明電源
- 檢查照明設備是否故障
```

**原因2：產品反光**
```
解決方法：
- 降低 exposure
- 降低 gain
```

---

### ❓ Q9：新料號和舊料號很像，參數可以直接複製嗎？

**答案**：可以，但仍需測試確認

```
建議流程：
1. 直接複製參考料號的參數
2. 不要修改，直接標記為已修改
3. 儲存到資料庫
4. 實際拍照測試
5. 如果效果不理想，再回來微調
```

**適用情況**：
- ✅ 同系列產品，只是尺寸微調
- ✅ 同顏色、同材質

**不適用情況**：
- ❌ 顏色差異大（如深色→淺色）
- ❌ 尺寸差異大（如大型→小型）

---

### ❓ Q10：參數值輸入後變成 0 或消失？

**A：可能輸入了不合法的值**

**常見原因**
```
1. 輸入了文字而不是數字
   ❌ exposure = "很亮"
   ✅ exposure = 6000

2. 輸入了超出範圍的值
   ❌ gain = 999
   ✅ gain = 12

3. 整數參數輸入了小數
   ❌ delay = 100.5
   ✅ delay = 100

4. 小數參數使用了逗號而不是句點
   ❌ threshold = 0,5
   ✅ threshold = 0.5
```

**解決方法**
```
1. 參考「參數說明」欄位的範圍
2. 使用正確的數值格式
3. 如果不確定，先複製參考區的值，再微調
```

## 校正工具問題

### ❓ Q11：校準工具開啟後沒有顯示影像？

**解決方法**：

**方法一：重新啟動程式**
```
1. 關閉所有視窗
2. 重新開啟主程式
3. 重新開啟參數設定視窗
```

---

### ❓ Q12：白色像素校正的標準差太大（> 0.5）怎麼辦？

**解決方法**：

**Step 1：檢查照明**
```
- 確認照明電源穩定
```

**Step 2：調整相機參數**
```
- 調整曝光時間避免過曝
- 降低增益減少雜訊
```

**Step 3：環境控制**
```
- 避免外部光源干擾
- 確保工作區域照明穩定
```

**Step 4：聯絡工程師**
```
如果以上方法都無法改善，請聯絡工程師檢查：
- 照明系統是否有問題
- 相機硬體是否正常
```

## 儲存與載入問題

### ❓ Q13：儲存時提示「資料庫鎖定」錯誤？

**A：資料庫正在被其他程式使用**

**原因**
```
- 主程式正在寫入資料庫
- 其他參數設定視窗同時開啟
- 資料庫檔案被防毒軟體掃描
```

**解決方法**
```
1. 稍等 5-10 秒後重試
2. 確認沒有開啟多個參數設定視窗
3. 如果持續發生：
   - 關閉主程式
   - 重新開啟參數設定視窗
```

---

### ❓ Q14：工作階段檔案載入後，參數沒有顯示？

**A：工作階段檔案可能損壞或格式錯誤**

**檢查方法**
```
1. 用記事本開啟工作階段檔案：
   bin\x64\Release\setting\param_session_{料號}.json

2. 檢查格式是否正確（應該是 JSON 格式）：
   {
     "targetType": "10215112360T",
     "sourceType": "10215112360T",
     "parameters": { ... }
   }

3. 如果格式錯誤：
   - 刪除該檔案
   - 重新設定參數並匯出
```

---

### ❓ Q15：儲存後測試發現參數不對，怎麼改？

**方法一：重新開啟參數設定視窗**
```
1. 關閉目前的參數設定視窗
2. 重新開啟目標料號的參數設定
3. 選擇相同的來源料號
4. 修改參數
5. 重新儲存
```

**方法二：直接在主視窗修改（不建議）**
```
⚠️ 只在緊急情況使用
- 直接在資料庫編輯（需要權限）
- 風險高，容易出錯
```

## 測試驗證問題

### ❓ Q16：「部分自動導入」功能找不到？

**A：檢查系統版本**

```
「部分自動導入」是新版本功能

查找位置：
在任何參數分類標籤（相機、位置、檢測、時間）的右上角
應該會看到 [🔄 部分自動導入] 按鈕

如果沒有：
- 確認系統版本 ≥ v1.1
- 或聯絡工程師更新
```

---

### ❓ Q17：部分自動導入後，參數看起來不對？

**A：自動導入的參數僅供參考，仍需人工確認**

```
建議操作：
1. 檢查自動導入的參數值
2. 根據實際需求微調
3. 重新測試驗證
4. 確認無誤後儲存

不要盲目相信自動導入的結果！
```

---

### ❓ Q18：時間參數的「根據OD自動計算」不準確？

**解決方法**
```
1. 使用自動計算值作為初始值
2. 實際測試觀察推料時機
3. 根據觀察結果手動微調：
   - 推早了（產品還沒到） → 時間增加 
   - 推晚了（產品已過頭） → 時間減少 
   增減多少自行估計
4. 重複測試直到準點
```
